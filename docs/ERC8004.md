# ERC-8004 Integration — SLAgent-402

## Overview

[ERC-8004 (Trustless Agents)](https://eips.ethereum.org/EIPS/eip-8004) extends Google's A2A protocol with an on-chain trust layer via three registries:

| Registry | Purpose | SLAgent-402 Mapping |
|----------|---------|-----------------|
| **Identity** | Agent registration (NFT-based) | Buyer/Seller register on-chain identities |
| **Validation** | Third-party verification of agent work | Receipt hash submitted for SLA compliance check |
| **Reputation** | Post-interaction feedback scores | Payout ratio recorded as reputation score |

## Architecture

```
Buyer Agent ─────┐
                  │  registerAgent()
                  ▼
          ┌─────────────────┐
          │ SLAOrchestrator │──── Identity Registry (agent IDs)
          │   (adapter)     │──── Validation Registry (receipt verification)
          │                 │──── Reputation Registry (SLA scores)
          └────────┬────────┘
                   │
          ┌────────▼────────┐
          │  SLASettlement  │ (existing settlement contract)
          └─────────────────┘
```

## Lifecycle Mapping

### 1. Agent Registration → Identity Registry

```
Buyer/Seller → SLAOrchestrator.registerAgent(agentURI)
             → Identity.register(agentURI) → agentId
             → Identity.setAgentWallet(agentId, wallet)
```

The `agentURI` is an off-chain JSON file describing the agent's capabilities, endpoints, and supported trust mechanisms. Example:

```json
{
  "type": "https://eips.ethereum.org/EIPS/eip-8004#registration-v1",
  "name": "SLAgent-402 Seller",
  "services": [
    { "name": "A2A", "endpoint": "https://seller.example.com", "skills": ["invoice-generation"] }
  ],
  "x402Support": true,
  "supportedTrust": ["reputation", "crypto-economic"]
}
```

### 2. Post-Settlement → Validation Registry

```
Gateway → SLAOrchestrator.recordValidation(requestId, receiptHash)
        → Validation.validationRequest(orchestrator, sellerAgentId, receiptHash)

Gateway → SLAOrchestrator.submitValidationResult(receiptHash, pass, "sla-compliance")
        → Validation.validationResponse(receiptHash, 100, "sla-compliance")
```

Validation response values:
- `100` = all validators passed (schema, tests)
- `0` = validation failed

### 3. Post-Finalization → Reputation Registry

```
Anyone → SLAOrchestrator.recordReputation(requestId)
       → Reputation.giveFeedback(sellerAgentId, score, 0, "sla", "settlement", "", receiptHash)
```

Score calculation: `score = (payout / maxPrice) * 100`

| Scenario | Payout | Score |
|----------|--------|-------|
| Fast + valid (full payout) | 100,000 | 100 |
| Slow but valid (partial) | 60,000 | 60 |
| Invalid (zero payout) | 0 | 0 |

## Contracts

| Contract | Location | Description |
|----------|----------|-------------|
| `IERC8004.sol` | `contracts/src/erc8004/` | Minimal interfaces for all 3 registries |
| `ERC8004Mocks.sol` | `contracts/src/erc8004/` | Mock implementations for testing/demo |
| `SLAOrchestrator.sol` | `contracts/src/erc8004/` | Adapter tying SLA settlement to ERC-8004 |

### SLAOrchestrator API

```solidity
// Register an agent (returns agentId)
function registerAgent(string calldata agentURI) external returns (uint256);

// Record receipt for validation (after settle)
function recordValidation(bytes32 requestId, bytes32 receiptHash) external;

// Submit validation result (gateway/validator only)
function submitValidationResult(bytes32 receiptHash, bool pass, string calldata tag) external;

// Record reputation (after finalize)
function recordReputation(bytes32 requestId) external;

// Lookup agent ID
function getAgentId(address agent) external view returns (uint256);
```

## Python Adapter

`gateway/app/erc8004.py` provides `ERC8004Adapter` with two modes:

- **Mock mode** (default): logs operations, tracks state in-memory
- **Chain mode**: submits transactions to SLAOrchestrator contract

Configure via environment variables:
```bash
export ORCHESTRATOR_CONTRACT_ADDRESS="0x..."  # enables chain mode
```

## Demo Flow (6 Steps)

1. **Register agents**: Buyer and seller each call `registerAgent(agentURI)`
2. **Link wallets**: Wallets linked automatically during registration
3. **Execute SLA call**: Normal SLAgent-402 flow (deposit → settle)
4. **Record validation**: Gateway calls `recordValidation(requestId, receiptHash)`
5. **Submit result**: Gateway calls `submitValidationResult(receiptHash, true, "sla-compliance")`
6. **Record reputation**: After finalization, call `recordReputation(requestId)`
